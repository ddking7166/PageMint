# JSX Pages

PageMint uses Hono JSX as server-only JSX. A page `render()` function returns either JSX or an HTML string.

The browser does not load React and PageMint does not hydrate pages by default.

## Page Responses

Pages can also return a PageMint response helper when they need a status code or headers:

```tsx
import { redirect, notFound, pageResponse } from '@pagemint/hono'

definePage({
  path: '/old-path',
  load() {
    return {}
  },
  render() {
    return redirect('/new-path', 301)
  },
})

definePage({
  path: '/articles/:id',
  load({ params }) {
    return findArticle(params.id)
  },
  render({ data }) {
    if (!data) return notFound(<html><body>Article not found</body></html>)
    return pageResponse(<html><body>{data.title}</body></html>, {
      status: 200,
      headers: { 'x-page-kind': 'article' },
    })
  },
})
```

Returning plain JSX or a string still produces a normal `200` HTML response.

## Client Scripts

PageMint server JSX is not hydrated by default. Add browser behavior explicitly with client modules:

```tsx
import { ClientScripts, defineClientManifest } from '@pagemint/hono'

const clientManifest = defineClientManifest({
  modules: {
    vendor: {
      scripts: [
        { src: '/static/vendor.js', type: 'classic', defer: true },
      ],
    },
    app: {
      dependsOn: ['vendor'],
      scripts: ['/static/build/assets/app.js'],
    },
  },
})

export function Layout({ children }) {
  return (
    <html>
      <head>
        <ClientScripts manifest={clientManifest} modules={['app']} />
      </head>
      <body>{children}</body>
    </html>
  )
}
```

String scripts are emitted as ES modules. Use a `ClientScript` object for classic scripts, CSP nonce, integrity, or custom attributes.

For local app code, point scripts at files generated by your bundler rather than raw source files.
